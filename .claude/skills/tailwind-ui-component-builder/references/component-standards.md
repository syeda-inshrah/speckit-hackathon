# Component Standards Reference

## Reusability
- Keep components stateless unless required
- Move API logic to pages, never inside components

## Props
- Always type props with interface or type alias
- Do not accept unnecessary props

## Structure
- Wrap component in a main container div
- Provide consistent padding
- Use semantic HTML elements

## Interaction
- Buttons -> <button>
- Links -> <a> or <Link>
- Forms -> <form>, <label>, <input>

## Accessibility
- aria-label for icon-only buttons
- Provide alt text for images
